# Prism Anisotropic Split Figures

This directory contains the primitive prism split-code figures that are not the
fully combined isotropic split.

The current implementation uses four primitive `Prism::mySplitCode` values:

- `0`: no split.
- `1`: axial split between the two end faces. This creates two child prisms.
- `2`: transverse split of the end-face polygon. This creates `nfold` children;
  for the root triangular prism, that is three quadrilateral-prism children.
- `3`: combined axial and transverse split. This creates `2*nfold` children;
  for the root triangular prism, that is the six-child isotropic case in
  `../isotropic/`.

In `Prism::setSplitCode()`, `split_mode == 0` may choose code `1`, `2`, or `3`
from the edge-length ratio and tolerance. `split_mode == 1` selects the
transverse code `2` when the end-face edges are large enough. `split_mode == 2`
selects the combined code `3` when the cell is large enough.

## Files

`prism_split_code_1_axial.tex` shows the axial split line scaffold. The
matching exploded wireframe source is
`prism_split_code_1_axial_exploded.tex`, and its GIF is regenerated by
`render_prism_split_code_1_axial_exploded_motion_yaw_wire.sh`.

`prism_split_code_2_transverse.tex` shows the transverse end-face split
scaffold. The matching exploded wireframe source is
`prism_split_code_2_transverse_exploded.tex`, and its GIF is regenerated by
`render_prism_split_code_2_transverse_exploded_motion_yaw_wire.sh`.

The GIF renderers use the same 81-frame assembled, explode, yaw, return, and
collapse sequence as the isotropic prism animation. The GIFs are generated
build artifacts and are intentionally not tracked in git.
